Analytics Platform Implementation

When faced with rising licensing and maintenance costs of our existing data platform, I accepted the task of identifying a new data storage platform for our project. This was a platform with capacity to store 1 years' worth of data while still delivering query performance that allowed analysts to meet daily work loads. This system stores 100TB of data and requires an interface that allows extracting data based on complex selection criteria. Not only did we need to query performance, we needed efficient extract load and transform capability.

The approach I took involved the following steps:/p>

  • Define requirements and constraints to include (money, size, power, cooling, ability to query, low side/high side--loading fast on H/S)
  • Define evaluation methods for systems and write an AoA -- what are the tactical advantages to an approach and what are the strategic advantages
  • Identify the challenges the future is likely to to bring
  • Building a team to execute the system evaluations
  • Come up with and execute a plan that proved efficient data loads and data extract
  • Choose a system that met constraints, then work with local resources to implement the system
