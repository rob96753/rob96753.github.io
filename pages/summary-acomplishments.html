Analytic Platform Implementation
Rising licensing and maintenance costs of our existing DBMS platform necessitated moving to a more cost effective solution. I proposed we investigate different data storage/query solutions for our analysis project. My proposal resulted in ...my leading the effort to identify system requirements and creating a team to evaluate alternatives for a new data storage platform that would host 100TB of data and support analysis through performant querying. The approach I took involved the following steps:
  • Define requirements and constraints to include (money, size, power, cooling, ability to query, "flat" architecture, low side/high side--loading fast on H/S)
  • Define evaluation methods for systems and write an AoA -- what are the tactical advantages to an approach and what are the strategic advantages
  • Recognize the challenges the future is likely to to bring to our project
  • Identify the candidate systems based on a paper study
  • Build a team and plan for system evaluations with real data
  • Organize and Execute the plan to quantify effacy of each candidate system
  • Choose a system that met constraints, then work with local resources to implement the system.
The system we purchased was a software only solution, that we ran on commodity Dell hardware. An issue we faced is the vendor suggested we use 10Gb as the interconnect. 10Gb hardware was very expensive at the time; We were able to accomplish equivalent performance running UDP over Infiniband-4 (IB-4).
Data Generation and Simmulation
The TMDS/MSAT Electronic Medical Records (EMR) system tracks service members' medical history through data integration from many internal and external data sources. The team doesn't have access to these systems, so I have created a number of tools to generate synthetic data files for training and testing. ... Ideally we would have copies of the same external applications so we can generate files to load longitudinal medical records into our EMR system. This hasn't been feasible due to resource constraints. I have developed a few tools to create synthetic EMR for use in training and testing:
  • Create a random patient demographics generator in Python to ensure PII isn't inadvertently entered in the systems.
  • Blood donation synthetic data generator to automatically load blood for training and testing (Python)
  • Data message patient demographics remapper used to refresh messages so scenarios can be retested (Python)
  • Synthetic message generator to map patients into representative longitudinal medical records used in training and testing (Java/JSON/HAPI).
  • Message to JSON parser to support scenario based, data driven testing of application business rules.

Project Planning
I led a team with the charter to create a web-based analytical platform supporting remote sites across the US. I knew a linear waterfall methodology wouldn't yield a relevant solution. I sought a better approach and Iterative development (SCRUM) looked like a great alternative...What was the approach Who were the actors -- DoD was a funding source along with DOE. We had a Product Owner -- who wasn't really able to articulate what the product should be. A project manager who helped track budgets, prepare status reports, and take care of project administration, developers, technical writer, tester, project admin, and PI. As PI, I identifying the requirements (usually working with the Product Owner), techical direction, and tasks we were taking on. I recruited the team members at PNNL and identified the approach for running the project (SCRUM) which we took the team, including the product owner, to training in Seattle. We were able to follow the SCRUM process, since team members had been trained in the process and bought into the approach. We were able to deliver a system that eventually replaced the thick-client Python GUI.
